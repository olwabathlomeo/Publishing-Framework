# Publishing Framework Style Guide

## Purpose

This document defines the writing and formatting standards used throughout this publishing framework.

---

# Headings

Use only one level 1 heading per file.

```markdown
# Chapter Title
```

Major sections use:

```markdown
## Section
```

Subsections use:

```markdown
### Subsection
```

---

# Writing Style

Write clearly.

Prefer short sentences.

Explain concepts before giving formal definitions.

Avoid unnecessary jargon.

---

# Mathematics

Display important equations on separate lines.

Define every symbol before using it.

Use consistent notation throughout the book.

---

# Figures

Every figure must have:

- Figure number
- Caption
- Source (if applicable)

---

# Tables

Every table must have:

- Table number
- Title
- Explanation

---

# Examples

Every example should contain:

- Problem
- Solution
- Discussion

---

# Exercises

Arrange exercises in increasing difficulty.

- Basic
- Intermediate
- Advanced
- Challenge

---

# References

Use one citation style throughout the project.

---

# File Naming

Use lowercase letters.

Separate words with hyphens.

Good:

chapter-one.md

Bad:

Chapter One.md

ChapterOne.md
