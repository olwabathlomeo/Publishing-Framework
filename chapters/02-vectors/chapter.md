# A Simple Working Example

The second chapter demonstrates how the book can hold both explanatory text and lightweight technical content. This example uses a short mathematical note to show that the pipeline can handle equations in Markdown.

Inline math: $\vec{v} = (1,2,3)$

You can also include displayed equations:

$$
\nabla \times \vec{F} = \left( \frac{\partial F_z}{\partial y} - \frac{\partial F_y}{\partial z}, \dots \right)
$$

This is intentionally simple, but it shows that the same source files can support both narrative writing and technical notation.
