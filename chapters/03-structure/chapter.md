# Book Structure

A publishing framework should be easy to navigate. This chapter explains the basic organization of the project so that authors can find the files they need quickly.

## Files and folders

The top-level structure is intentionally simple:

- `chapters/` holds the main content
- `frontmatter/` stores title and metadata files
- `templates/` contains reusable layout patterns
- `output/` is generated by the build process

Keeping these areas separate makes it easier to maintain the project over time.
